# AI-native спецификация

Что делает решение AI-native: ИИ-агент встроен **в сам продукт** (доработка на естественном языке), а разработка велась **через ИИ-агентов** с человеком-директором.

## Модель ИИ-ассистированной разработки
- **Владелец** — директор: ставит задачи, ревьюит, принимает/отвергает.
- **Claude Code** — оркестратор: бэкенд, инфраструктура, интеграция, ревью, безопасность.
- **Codex CLI** — суб-агент: дизайн и фронтенд (по точным ТЗ), генерация изображений.
- **Встроенный codex-агент** — автономная доработка продукта из UI (см. ниже).

## Встроенный автономный агент («Чат с разработчиком»)
Доступен только владельцу (`username === 'spellful'`). Архитектура решает ограничение: приложение в контейнере не может запускать хостовый `codex` и перезапускать контейнеры. Поэтому — **паттерн хост-воркера**:

```
UI (spellful) → POST /api/admin/chat → таблица admin_tasks (очередь)
   → systemd-воркер на хосте: codex exec -C /opt/tashkent (sandbox=workspace-write)
   → git commit/push → при правке бэкенда: docker restart tashkent
   → статус+вывод в admin_tasks → поллинг в UI
```
- **Scope/безопасность:** codex ограничен каталогом проекта (sandbox), задачам предпослан системный контекст (`/root/tashkent_agent_context.txt`). Это эффективно RCE через вход `spellful` — осознанный компромисс (ADR-0004).
- **Анализ фото:** изображения из чата сохраняются в `data/admin_uploads/` (внутри проекта); codex читает их с диска и анализирует — без изменения воркера.

## Генерация и ревью ассетов
- Иллюстрации лендинга сгенерированы `codex image_gen` (GPT Image), работающим даже на ChatGPT-авторизации.
- **Анти-артефактный приём:** только предметные сцены (стол, шары, сукно, свет) — **без людей, рук, лиц и текста** — чтобы исключить классические дефекты («лишние пальцы», кривые надписи). Усиленные негативные промпты.
- **Ревью-петля:** каждую картинку отсматривает Claude (визуально) на артефакты; плохую — перегенерировать с уточнённым промптом. Затем оптимизация PNG→JPG. См. ADR-0005.

## Усвоенные ограничения ИИ-инструментов (важно для оценки обучения)
- Keyless-генерация изображений (Pollinations) — за paywall/лимитом, ненадёжна с сервера.
- ChatGPT-oauth-токен codex не годится для публичного OpenAI image-API; но встроенный `image_gen` доступен.
- Sandbox codex ограничивает запись каталогом — отсюда паттерн хост-воркера для привилегированных операций.
