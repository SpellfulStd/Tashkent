# Продуктовая спецификация

> Источник истины по правилам счёта — `public/app.js` (вычисление состояния из событий). Здесь — намерение.

## Домен
Учёт партий в русский бильярд (пирамида) — «Ташкент». Группа игроков ведёт партию; события фиксируются с телефонов в реальном времени; партии объединяются в серии; накапливается статистика.

## Сущности (схема — `tashkent_schema.sql`)
- **players** — игрок: `name`, опц. привязка к аккаунту (`account_sub`). Гость = без привязки.
- **accounts** — аккаунты Keycloak, входившие в приложение (`sub`, `username`, `email`).
- **series** — серия партий: `status` (active|finished).
- **games** — партия: `series_id`, `target_balls`, `status`, `final_scores` (JSONB), `winner_player_id`, `points_leader_player_id`.
- **game_players** — состав партии и очерёдность (`position`), со снимком имени (история не теряется при удалении игрока).
- **game_events** — append-only события партии (`seq`, `player_id`, `type`, `created_by_sub`).
- **session** — сессии (connect-pg-simple). **admin_tasks** — очередь задач админ-чата (см. `ai-native.md`).

## Правила счёта (домен)
- Победа: первым набрать `target_balls` (настраивается при старте).
- **Обычный шар** (`pocket_regular`): +1 шар, +1 очко, ход остаётся.
- **Дуплет** (`pocket_duplet`): +1 шар, +2 очка, ход остаётся.
- **Штаны**: +2 шара, +3 очка, ход остаётся.
- **Штраф** (`penalty`): −1 шар, ход переходит (минусы — медным цветом, не красным).
- **Передача хода** (`set_turn`): явная смена хода. (**Промах** `miss` исторически передавал ход; по решению владельца убран из UI и лога, но прошлые события сохранены для корректного пересчёта.)
- **Дурак** (`duraks`): отдельный счётчик в `final_scores`.
- **Золотой шар**: условие — у игроков 15 шаров суммарно, по 5 у каждого (для 3 игроков) — добавлено владельцем через админ-чат.
- **Взаимозачёт**: очки отображаются со взаимным зачётом между игроками.

## Ключевые фичи
- **Real-time** мультиустройство: одна партия — несколько телефонов, синхронно.
- **Авторизация/регистрация** (Keycloak); новый пользователь сразу становится игроком.
- **Автозаход** в активную игру/серию, если игрок пользователя уже в составе (иначе — просмотр).
- **Гости**: игрок без аккаунта; за него отмечает любой участник состава.
- **Объединение профилей**: непривязанного игрока можно слить в свой профиль (перенос истории).
- **Видимость по составу**: каждый видит только свои игры; владелец (`spellful`) — все.
- **Серии, история (разные режимы отображения), статистика игроков.**
- **Админ-чат** (только владелец) — доработка продукта на естественном языке.
- **Донаты** на всех страницах кроме авторизации + CTA-благодарность на итогах.
