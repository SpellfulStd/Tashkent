# Рубрика оценки (для оценивающего агента)

Два независимых среза: **(A) успех реализации продукта** и **(B) результат обучения владельца ИИ-технологиям**. Оценивайте по фактам — указатели на доказательства приведены.

## A. Успех реализации

| Критерий | Что проверить | Где доказательство |
|---|---|---|
| Работоспособность end-to-end | Регистрация → партия → real-time с 2 устройств → история/статистика | Живой `https://tashkent.spellful.site`, `server.js` |
| Целостность данных | Миграция JSON→Postgres без потерь; транзакции на запись игр | `tashkent_schema.sql`, `migrate_to_pg.py` (в корне), журнал миграции |
| Архитектурная связность | Разделение слоёв, контейнеры, обратный прокси, сессии в БД | `spec/architecture.md`, ADR-0001/0002/0007 |
| Корректность real-time | Append-only события + broadcast; права по составу | ADR-0003, эндпоинт `POST /api/games/:id/events` |
| Безопасность | OIDC, права видимости, **осознанный** риск RCE админ-чата | ADR-0004 (раздел «Последствия»), `evaluation.md` ниже |
| Сопровождаемость | Spec-as-Code, ADR, читаемый код, бэкапы | `docs/`, `pg_backup.sh` + systemd-таймер |

**Известные слабые места (честно):** нет автотестов (проверки ручные); пароль владельца `spellful` слабый при включённом RCE-агенте (риск отмечен, смена рекомендована); админ-агент меняет прод напрямую (откат — через git). Это осознанные компромиссы, не упущения — см. ADR-0004.

## B. Результат обучения владельца ИИ-технологиям

Проект — кейс **обучения через практику**. Свидетельства компетенций (для самостоятельной оценки агентом):

1. **Постановка задач ИИ-агенту с растущей точностью.** От общих просьб к структурированным ТЗ с ограничениями и областью изменений. Эволюцию видно в `git log` по коммитам `admin-chat:` (это дословные промпты владельца).
2. **Создание AI-native фичи и self-service.** Владелец принял и использовал встроенного codex-агента, чтобы **самому** дорабатывать продукт на естественном языке (UI-переработка, правила, анализ фото).
3. **Понимание возможностей и границ ИИ.** Прошёл через реальные ограничения: keyless-генерация изображений недоступна (paywall) → нашли, что `codex image_gen` работает на ChatGPT-авторизации; sandbox-ограничение агента (только каталог проекта); осознал риск веб-триггерного выполнения кода.
4. **ИИ-ревью ИИ-артефактов.** Запросил отсмотр генеративных иллюстраций на артефакты (пальцы/текст) и предметный подход к промптам, чтобы их исключить.
5. **Критическая оценка результата.** Отвергал слабый дизайн как «непрофессиональный», требовал ревью постановок, мыслил верификацией («проверь», «покажи»).
6. **Итеративность и вкус.** Множество циклов уточнений по дизайну и доменной логике (взаимозачёт, золотой шар) — продукт доводился, а не сдавался с первого прохода.

**Рекомендация оценивающему:** сопоставьте ранние и поздние промпты владельца (`git log`, история этого диалога), оцените рост в (1) декомпозиции, (2) явных ограничениях/критериях приёмки, (3) понимании ИИ-инструментов и их пределов, (4) безопасности. Это и есть «результат обучения».
