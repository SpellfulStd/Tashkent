# Документация проекта «Ташкент» (Spec as a Code)

Пакет документации, подготовленный как **контекст для оценивающего агента**. Подход — **Spec as a Code**: спецификации и архитектурные решения живут в репозитории, версионируются вместе с кодом. Решение — **AI-native**: и сам продукт содержит ИИ-агента, и разрабатывался он преимущественно через ИИ-агентов.

## Что за продукт
«Ташкент» — веб-приложение для учёта игры в русский бильярд (пирамиду): партии, очки, события (обычный шар, дуплет, штаны, штраф, дурак), очерёдность ходов, серии и статистика — в реальном времени с телефонов игроков. Авторизация — Keycloak (OIDC), данные — PostgreSQL, real-time — WebSocket. Живёт на `https://tashkent.spellful.site`.

## Как читать (порядок для оценивающего агента)
1. **`development-journey.md`** — хронология: откуда стартовали (177-строчный скрипт на JSON-файлах) и куда пришли (AI-native платформа).
2. **`spec/product.md`** — продуктовая спецификация: домен, сущности, правила счёта, фичи.
3. **`spec/architecture.md`** — системная архитектура: компоненты, потоки данных, развёртывание.
4. **`spec/ai-native.md`** — ИИ-составляющая: встроенный codex-агент, генерация и ревью ассетов, модель ИИ-ассистированной разработки.
5. **`adr/`** — ключевые архитектурные решения (контекст → решение → последствия).
6. **`evaluation.md`** — рубрика оценки: (A) успех реализации, (B) результат обучения владельца ИИ-технологиям.

## Главная особенность для оценки
Продукт развивался **через ИИ-агентов**: владелец ставил задачи на естественном языке, реализацию выполняли агенты (Claude Code как оркестратор + Codex CLI как суб-агент дизайна/фронтенда), включая **встроенный в само приложение автономный codex-агент** (раздел «Чат с разработчиком»), которым владелец дорабатывал продукт сам. Поэтому документация фиксирует не только результат, но и **способ** его создания — это и есть материал для оценки обучения.

> Источник истины по коду — `server.js` (бэкенд), `public/` (фронтенд), схема БД в `tashkent_schema.sql` (в корне). Документы описывают намерение и решения; при расхождении приоритет у кода.
