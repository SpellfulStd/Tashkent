# ADR-0003: Real-time через WebSocket (notify-and-refetch)

**Статус:** принято · Этап 2

## Контекст
Требование владельца: каждый игрок управляет одной партией со своего телефона, изменения видны почти мгновенно. Прежняя модель (PUT всего объекта игры) при одновременных действиях затирала бы состояние.

## Решение
**Append-only события + WebSocket-уведомления.** Действие = `POST /api/games/:id/events {type, playerId}` дописывает событие (порядковый `seq`), право проверяется по составу (игрок пользователя должен быть в партии). Сервер рассылает по WS лёгкое сообщение `{type:'gameUpdated', gameId}` / `{type:'activeChanged'}`; клиенты перезапрашивают игру и пересчитывают счёт из событий. WS-соединение авторизуется сессией на upgrade.

## Последствия
- (+) Нет гонок «последний PUT побеждает»; устойчиво к нескольким устройствам; простой и надёжный протокол.
- (+) Расчёт очков — на клиенте из событий; сервер хранит факты, не дублирует доменную логику.
- (−) На каждое уведомление — повторный GET игры (для нескольких игроков несущественно).
- Альтернатива (стриминг каждого изменения состояния по WS) отклонена как избыточная сложность для масштаба «компания друзей».
