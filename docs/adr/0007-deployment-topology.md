# ADR-0007: Топология развёртывания (Docker + nginx + хост-воркер)

**Статус:** принято · Этапы 1–3

## Контекст
Сервис размещён на одном хосте, где уже работают другие нагрузки. Нужны изоляция, HTTPS, доступ контейнеров к Postgres/Keycloak, безопасные привилегированные операции для админ-агента, и сохранность живых данных при обновлениях кода.

## Решение
- **Контейнеры в общей сети `tashkent-net`:** приложение, `postgres`, `keycloak` общаются по именам; наружу порты не публикуются (кроме `127.0.0.1` для nginx). `--restart unless-stopped`.
- **nginx на хосте** — единая точка TLS (Let's Encrypt, автопродление) и reverse-proxy; поддомены `tashkent` и `auth`; проксирование WebSocket.
- **Код+данные в git-клоне `/opt/tashkent`** (монтируется в контейнер). Живые данные защищены `git update-index --skip-worktree`, чтобы `git pull` их не затирал; `node_modules`, `data/admin_uploads` — в `.gitignore`.
- **Хост-воркер** (systemd) выполняет привилегированные операции (codex, git push, `docker restart`), недоступные из контейнера — см. ADR-0004.
- **Эксплуатация:** бэкап Postgres (systemd-таймер), swap 4 ГБ, доступ к GitHub через `gh` (push агента).

## Последствия
- (+) Изоляция, воспроизводимость, единый TLS, безопасный канал к Keycloak (host-gateway), сохранность данных при обновлениях.
- (−) Несколько движущихся частей (контейнеры + nginx + systemd-воркер + cron-подобные таймеры) — выше эксплуатационная сложность.
- (−) Один хост — единая точка отказа; HA не предусмотрено (адекватно масштабу).
