## Хронология задач

### 2026-06-07 — первая сессия

**Задача:** Найти правила «Ташкента» и сделать веб-страницу учёта шаров.

**Проблема:** Канонических правил в открытых источниках нет — это народная разновидность
русской пирамиды. Правила уточнены у заказчика.

**Правила (продиктованы Виталием):**

1. Кол-во шаров до победы N задаётся при старте (по умолчанию 6).
   Зависит от числа игроков:
   - 2 игрока → 8
   - 3 игрока → 6
   - 4 игрока → 5
   - 5+ → 4
2. Дуплет = 2 очка, 1 шар в зачёт; Штаны = 3 очка, 2 шара; Обычный = 1 очко, 1 шар.
3. Хранение долгое (история игр), игроков переиспользовать между играми.

**Первая корректировка (в той же сессии):**

> «Забитые шары должны вычитаться из очков предыдущего игрока.
> Т.е. обычный +1 шар, +1 очко, −1 очко предыдущему игроку
> (если бил 2й, то минус 1му; если бил 3й, то минус 1му)»

**Вторая корректировка (та же сессия):**

> «Нет. Я ошибся. Предыдущий именно по номеру, а не последний забивший.
> Если игроков три, то предыдущим для 1го будет 3й игрок.»

→ Реализован алгоритм с фиксированным порядком: `prevIdx = (idx - 1 + n) % n`.

**Добавлены серии игр** — группировка партий с итоговым зачётом (победы + очки).

**Настроен serveo-туннель** для доступа с телефона: `ssh -R 80:localhost:7777 serveo.net`

### 2026-06-13 — вторая сессия

**Задача 1:** Добавить возможность продолжить игру после 6-го шара.

> «После победы игрок может забить ещё несколько шаров.
> После 6го шара можно не заканчивать игру, а сделать кнопку «завершить партию».»

**Решение:** Убрано авто-завершение при достижении targetBalls.
Вместо этого:
- При достижении N шаров → устанавливается `game.firstWinnerId`, показывается баннер
- Игра продолжается (все кнопки активны)
- Кнопка **«Завершить партию»** финализирует игру с текущими очками
- `winnerId` = `firstWinnerId` (первый достигший N шаров)
- `pointsLeaderId` = лидер по очкам на момент нажатия кнопки

**Задача 2:** Добавить историческую игру «07.06.26, 15:35 · до 6 шаров · 55м» к серии.

**Решение:** Идентифицирована как игра `e239873d` (createdAt `2026-06-07T11:35` UTC =
15:35 Ташкент UTC+4, длительность 55м). Добавлен `seriesId` в `data/games.json`.

## Технические решения и почему

| Решение | Причина |
|---|---|
| Node без зависимостей | Простота деплоя, не нужен npm install |
| JSON-файлы вместо БД | «Можно прям локально в файлах» — слова заказчика |
| Replay всех событий в `computeState` | Undo реализуется тривиально (pop последнего события) |
| Hash-based роутер | Одностраничное приложение без серверного роутинга |
| `firstWinnerId` отдельно от `winnerId` | Позволяет продолжать игру после достижения цели |
